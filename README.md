# transaction-activity
These files were originally created while comparing different methods of recording transaction activity and execution results. Instead of building a complete framework, the repository gradually became a place for keeping small experiments that proved useful more than once.

Some examples include market-related measurements that appear during testing sessions. Metrics such as fully diluted valuation are occasionally stored alongside transaction information because they help provide additional context when reviewing historical runs.

Another recurring topic is circulation. Several utilities were written simply to observe how values move through different stages of an execution process and how recorded information changes over time.

A few experiments were designed to compare alternative approaches and identify cases where one implementation was outperforming another. The goal was not benchmarking in the traditional sense, but rather understanding which ideas produced clearer behavior and easier maintenance.

Most files remain intentionally independent. They represent separate experiments performed at different times, which explains why the coding styles and implementation choices vary throughout the repository.
